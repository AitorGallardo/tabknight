## 0.15.3

Released in commit [`72f9dd6`](https://github.com/aitorgallardo/tabknight/commit/72f9dd692f0f166cd016961299cb3348654062c2)

### Added

- support for Chrome-native tabs in navigator results, including pages such as `chrome://extensions/`
- restricted-page fallback flow that opens a temporary adjacent extension tab when in-page rendering is blocked
- screenshot-backed contextual fallback rendering using a blurred capture of the source tab
- halftone and vignette visual treatment for the fallback navigator surface
- storage-backed context handoff between the background worker and fallback UI

### Changed

- preserved overlay-first behavior for `Cmd+K` and `Cmd+Ctrl+T` on normal `http/https` pages
- unified the main keyboard-driven navigator around the React Arc-style shell while preserving glossy translucency
- removed artificial result limits so the full list of matching tabs is visible
- rewrote README sections to document architecture, permissions, keyboard flows, and constraints

### Fixed

- active result now scrolls into view during keyboard navigation in both the content-script overlay and the standalone/popup navigator
- temporary fallback navigator now self-cleans by closing after selection and restoring the origin tab on `Escape`

## 0.13.0

Released in commit [`7c41d7a`](https://github.com/aitorgallardo/tabknight/commit/7c41d7afd7cd1a0abfbd2a35d6c2a3f7e3990db8)

### Added

- production-ready Arc-inspired in-page tab navigator content script
- background runtime bridge for querying tabs, activating tabs, and opening search/URL queries in new tabs
- keyboard routes for `Cmd+K` and `Cmd+Ctrl+T`
- robust command-open path with reinjection fallback on supported pages
- popup Arc-style navigator view as the default popup entry
- content script registration in the build and manifest pipeline
- required `scripting`, `windows`, and host permissions for the navigator flow
- project-level versioning and UI rules in `AGENTS.md`

### Changed

- improved save and close flows, including preserved favicon context and stronger post-close restore affordances
- replaced legacy SVG icon assets with PNG variants for extension packaging
- rewrote README to reflect the implemented architecture and release policy

### Fixed

- cross-window activation reliability by resolving the tab, focusing its window, activating the tab, and highlighting its index

## Initial Commit

Created in commit [`4ffed57`](https://github.com/aitorgallardo/tabknight/commit/4ffed571a5340bdb2b2c2915f8c2484947aad309)

### Added

- initial TabKnight Chrome extension scaffold
- React 18 + TypeScript + Tailwind CSS + shadcn/ui popup application
- bookmark-backed session save, restore, and close flows
- base Chrome API wrappers for tabs, bookmarks, and window operations
- MV3 manifest, build pipeline, icons, and extension packaging assets
- foundational hooks and popup components for tab selection and domain grouping

### Notes

- this predates the formal `0.12.0` baseline policy documented later in `AGENTS.md`
- it established the project as a keyboard-first Chrome tab manager before the Arc-style navigator work landed
